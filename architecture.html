Info

This section is intended to give a brief insight into the architecture of the entire system. On the whole, the system consists of the following three big components:

  • Backend
  • Android Application
  • Locking System

Simplified Architecture:

In the simplified representation of the system the 3 basic components can be seen. The blue arrows indicate communication via NFC. The other arrows represent communication between the individual components and the server via the Internet. The communication is based on HTTPS. During communication between server and Raspberry or server and app, system-specific data is exchanged.
With the Android application a new chip can be written and read. When the chip is written, a new authorization for the respective lock is created. Reading the chip serves to determine the owner of a key.

Challenges


The main challenges to the security of the system start with the multitenancy. This means that there are several clients on the same server, i.e. customers or principals, and that they can operate the system without having mutual access to their data, user administration and so on. This also includes the privacy of the individual users.

A further important point is the safety of the system against disturbing influences, or also environmental influences, from outside. It should not be possible for the attacker to bypass the lock by violent actions.

The NFC chip cards used also play an important role. Chip cards are to be used that guarantee secure data encryption.

The connection between the individual components (server, lock, app) must also be secure and therefore not vulnerable.

This system has tried to overcome all these security challenges.
