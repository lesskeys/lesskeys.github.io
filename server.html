LessKeys-Server

Info

The central component and data storage of the system is the backend, via which all communication of individual system elements takes place. The server functions as the actual central communication interface of the system.
It is implemented as a REST API and thus provides various services for the individual components. As usual with REST, HTTP is used here as the communication method. In our special case HTTPS is used to guarantee confidential data traffic. All data is stored in an SQL-based database.

Technical part

The server application was implemented in Java using the Spring Framework. The SQL-based database, more precisely a MySQL database, is controlled by the Java Persistance API (JPA). In this database all data of the system are stored. As webserver we use Apache Tomcat, because we need an ApplicationServer that allows us to host Java Servlets. Apache Maven is used for build and dependency management.
