System Architecture

This section is intended to give a brief insight into the architecture of the entire system. On the whole, the system consists of the following three big components:

  • Backend
  • Android Application
  • Locking System

Simplified Architecture:

In the simplified representation of the system the 3 basic components can be seen. The blue arrows indicate communication via NFC. The other arrows represent communication between the individual components and the server via the Internet. The communication is based on HTTPS. During communication between server and Raspberry or server and app, system-specific data is exchanged.
With the Android application a new chip can be written and read. When the chip is written, a new authorization for the respective lock is created. Reading the chip serves to determine the owner of a key.
